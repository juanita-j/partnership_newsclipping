# 뉴스클리핑 메일 미수신 시 확인 체크리스트

## 1. 스케줄은 GitHub에서만 실행됩니다

- **9시~19시 KST 1시간마다(11회)** 발송은 **GitHub Actions** 스케줄로 동작합니다.
- 로컬 PC에서는 10시에 자동 실행되는 스크립트가 없습니다.
- 해당 리포지토리가 **GitHub에 푸시되어 있고**, **Actions 탭에서 워크플로가 활성화**되어 있어야 합니다.

## 2. GitHub Actions에서 확인할 것

1. **GitHub 저장소** → **Actions** 탭 이동
2. 왼쪽에서 **"Send News Mail"** 워크플로 선택
3. **해당 시각(KST 9~19시 매시 5분) 전후** 실행 기록 확인
   - 예: KST 10:05 = UTC 01:05 (cron: `5 1 * * *`)
   - 스케줄 실행은 최대 10~15분 지연될 수 있음
4. 실행이 **있었다면**: 각 단계(Collect news → Summarize with LLM → Send mail) 로그 확인
   - **Collect news** 실패: 네이버 API 키/할당량, 네트워크
   - **Summarize with LLM** 실패: OPENAI_API_KEY, 요청 한도
   - **Send mail** 실패: GMAIL_APP_PASSWORD, 2단계 인증/앱 비밀번호
5. 실행이 **없었다면**: 해당 브랜치에 `.github/workflows/send-news-mail.yml`이 있는지, Actions가 비활성화되지 않았는지 확인

## 3. 메일이 "스킵"되는 경우 (에러 없이)

- **요약 항목 0건**  
  정기(scheduled) 모드에서는 **직전 슬롯 이후** 기사만 수집합니다.  
  - 10시 run: 전날 18:00 ~ 당일 10:00 기사만  
  이 구간에 파트너사+임원인사/조직개편 조건에 맞는 기사가 없으면 `news_summary.json` items가 0건이 되고, **"요약 항목 0건. 메일 발송 스킵"**으로 메일을 보내지 않습니다.

- **동일 내용 24시간 이내 이미 발송**  
  본문 해시가 같고 마지막 발송이 24시간 이내면 **"동일 내용이 24시간 이내 이미 발송됨. 발송 스킵"**으로 다시 보내지 않습니다.

## 4. 수동으로 한 번 보내고 싶을 때

- **GitHub**: Actions → Send News Mail → **Run workflow** (workflow_dispatch)  
  → 당일 00:00 KST ~ 실행 시각까지의 기사로 수집·요약·발송합니다.

- **로컬에서 파이프라인 테스트** (환경 변수 설정 후):
  ```bat
  set REQUEST_SCOPE=today
  python send_exec_news_timed.py
  python summarize_exec_news_llm.py
  python send_email_from_json.py
  ```
  또는 `FORCE_SEND=1`로 동일 내용이라도 재발송:
  ```bat
  set FORCE_SEND=1
  python send_email_from_json.py
  ```

## 5. 필수 Secrets (GitHub 저장소 설정)

- `NAVER_CLIENT_ID`, `NAVER_CLIENT_SECRET`
- `OPENAI_API_KEY`
- `GMAIL_APP_PASSWORD`, `GMAIL_SENDER`, `GMAIL_TO`

위 값이 없거나 만료되면 해당 단계에서 실패합니다.
