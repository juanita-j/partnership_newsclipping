# 파트너사 뉴스 클리핑 — 인수인계 가이드

이 문서는 **이 프로그램을 처음 보는 사람**이 무엇을 하는 소프트웨어인지 이해하고, **Cursor**에서 저장소를 열어 **설정·코드를 안전하게 수정**할 수 있게 정리했습니다.

---

## 1. 한 줄로 요약

**지정한 파트너사 이름으로 뉴스를 검색 → 관련 없는 기사를 걸러냄 → AI(또는 규칙)로 요약 → HTML 메일로 묶어 SMTP로 발송**하는 **배치 프로그램**입니다.  
로컬에서 `python run_batch.py`로 돌리거나, **GitHub Actions**에서 같은 스크립트를 실행합니다.

---

## 2. 사전 준비 (처음 세팅하는 사람)

| 항목 | 설명 |
|------|------|
| **Python** | 3.11 권장 (Actions도 3.11). `python --version` 확인 |
| **Git** | 저장소 클론·푸시용 |
| **Cursor** | VS Code 기반 에디터. 이 폴더를 **폴더로 열기(File → Open Folder)** |
| **계정·키** | 네이버 검색 API, LLM(OpenAI 또는 Anthropic), Gmail(SMTP 앱 비밀번호) 등은 **코드에 넣지 말고** 환경변수 또는 GitHub Secrets에만 둡니다 |

---

## 3. Cursor에서 프로젝트 열기

1. GitHub에서 저장소를 **Fork**하거나, 권한이 있으면 **Clone**합니다.  
   ```bash
   git clone https://github.com/<조직또는계정>/partnership_newsclipping.git
   cd partnership_newsclipping
   ```
2. **Cursor** 실행 → **File → Open Folder** → 위 `partnership_newsclipping` 폴더 선택.
3. 터미널은 Cursor 하단 **Terminal**에서 열면 됩니다 (작업 디렉터리가 프로젝트 루트인지 확인).

**팁:** Python 인터프리터가 프로젝트 루트를 인식하도록, 가상환경을 쓰는 경우 Cursor에서 해당 venv의 Python을 선택합니다 (Command Palette → `Python: Select Interpreter`).

---

## 4. 로컬에서 한 번 돌려보기

```powershell
cd partnership_newsclipping
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

환경변수(네이버, LLM, SMTP 등)를 README와 동일하게 설정한 뒤:

```powershell
python run_batch.py --dry-run
```

- **`--dry-run`**: 수집·필터·요약·HTML 생성까지 하고 **메일은 보내지 않음** (안전하게 동작 확인).
- **`--no-llm`**: API 없이 규칙 기반 요약만 쓰고 싶을 때.

실제 발송:

```powershell
python run_batch.py
```

## 5. 처리 흐름 (데이터가 어떻게 흐르는지)

아래는 `run_batch.py`가 하는 일의 순서입니다.

```text
[1] storage/last_send.py
      → 직전 메일 발송 성공 시각 이후만 수집 (없으면 최근 7일 등 폴백)

[2] collectors/
      → 네이버 뉴스 API(주) + Google News RSS(보조)로 기사 수집
      → 각 기사는 partner_id(어떤 파트너 검색으로 걸렸는지)를 가짐

[3] filters/keyword_filter.py 등
      → 블로그 제외, 키워드·제목 규칙, 파트너별 제외 로직

[4] filters/partner_cap.py
      → 파트너별 최대 건수 등

[5] summarizers/
      → LLM으로 3~7줄 요약 (설정: config/summarizer.yaml)

[6] dedup/, compose/merge_same_topic.py
      → 중복·유사 기사 정리

[7] compose/html_composer.py + templates/email.html
      → 국내/글로벌 섹션, 회사별 HTML 메일 본문

[8] sender/send.py
      → SMTP 발송. 성공 시 run_batch가 storage에 발송 시각 기록
```

**수정할 때:** “어디를 고치면 되지?”는 대부분 **어느 단계를 바꾸고 싶은지**에 따라 결정됩니다 (아래 7절 참고).

## 6. 폴더·파일 지도 (무엇이 어디 있나)

| 경로 | 역할 |
|------|------|
| **`run_batch.py`** | 전체 파이프라인 진입점. 여기서 import되는 모듈 순서가 실행 순서와 대응됩니다. |
| **`config/partners.yaml`** | 추적할 파트너 목록. 각 항목 `id`, 검색용 `names` (여러 별칭 가능). |
| **`config/sections.yaml`** | 메일에서 **국내/글로벌** 구분, `domestic_groups` / `global_groups`의 **머릿말·그룹·순서**. `domestic` / `global` 플랫 리스트는 그룹 미사용 시 폴백. |
| **`config/keywords.yaml`** | “주요 뉴스”로 볼 만한 키워드 그룹 (1개 이상 포함 시 통과 쪽에 사용). |
| **`config/group_subsidiary_labels.yaml`** | 메일 본문 `[머릿말]`에 쓸 계열사명 매칭 후보. |
| **`config/partner_article_caps.yaml`** | 파트너별 최대 기사 수 등. |
| **`config/partner_business_relevance.yaml`** | 브랜드만 같고 사업 무관한 오탐(병원 상호 등) 제외 규칙. |
| **`config/sender.yaml`** | SMTP 기본값 (환경변수가 우선). |
| **`config/summarizer.yaml`** | LLM 제공자·모델. |
| **`config/dedup.yaml`** | 중복 제거 관련 설정. |
| **`filters/keyword_filter.py`** | 제외 키워드, 파트너별 특수 제외(에어비앤비, 바이트댄스, 오픈AI, 스포츠, 비유 제목 등)가 **코드에 많이 모여 있음**. |
| **`filters/business_relevance.py`** | YAML 기반 무관 기사 제외. |
| **`filters/partner_cap.py`** | 파트너별 상한. |
| **`compose/html_composer.py`** | HTML 생성 로직 (영문 머릿말, 기타 회사 분리 등). |
| **`compose/templates/email.html`** | Jinja2 HTML 템플릿. |
| **`compose/bracket_label.py`** | `[회사표시명]` 라벨 결정. |
| **`storage/last_send.py`** | `data/last_send_at.txt` 읽기/쓰기 (로컬, gitignore 대상). |
| **`.github/workflows/news-clipping.yml`** | Actions에서 `run_batch.py` 실행, Secrets 주입, `last_send_at` 캐시. |
| **`tests/`** | 소규모 스크립트형 테스트 (`PYTHONPATH=.` 필요할 수 있음). |

## 7. “이럴 때 어느 파일을 보나” 빠른 표

| 하고 싶은 일 | 우선 볼 곳 |
|--------------|------------|
| 파트너 추가/삭제, 검색어 변경 | `config/partners.yaml` |
| 메일 섹션 순서·그룹 이름·기타 회사 묶음 | `config/sections.yaml` |
| 어떤 키워드가 있어야 통과하는지 | `config/keywords.yaml` |
| 특정 단어 나오면 기사 제외 | `filters/keyword_filter.py` (`EXCLUDE_KEYWORDS`, 파트너별 함수) |
| 메일 표 머릿말·레이아웃 | `compose/html_composer.py`, `compose/templates/email.html` |
| `[삼성전자]` 같은 대괄호 라벨 | `config/group_subsidiary_labels.yaml`, `compose/bracket_label.py` |
| 요약 모델/API | `config/summarizer.yaml`, 환경변수 |
| 발송 주소·SMTP | `config/sender.yaml`, 환경변수, GitHub Secrets |
| Actions 실행 주기·수동 실행 | `.github/workflows/news-clipping.yml` |

## 8. GitHub Actions와 메일 반영

- **코드를 수정한 뒤 메일에 반영하려면** 변경분을 **`main`에 커밋·푸시**해야 합니다. Actions는 원격 저장소의 커밋을 체크아웃합니다.
- **푸시만으로 자동 발송되지는 않습니다.** (의도된 설정)  
  발송이 필요하면 저장소 **Actions → Partner News Clipping → Run workflow** 로 수동 실행하거나, 예약(cron)을 기다립니다.
- **Secrets** (네이버, OpenAI/Anthropic, SMTP 등)는 저장소 **Settings → Secrets and variables → Actions** 에서 관리합니다. 이름은 `news-clipping.yml`의 `${{ secrets.XXX }}`와 일치해야 합니다.
- Actions 러너는 매번 초기화되므로, **직전 발송 시각**은 워크플로의 **cache**(`data/last_send_at.txt`)로 넘깁니다.

## 9. Cursor로 수정할 때 습관 (추천)

1. **작은 변경**부터 하고 `python run_batch.py --dry-run`으로 확인.
2. **`config/*.yaml`** 은 문법(들여쓰기, 하이픈 리스트) 깨지면 전체가 실패할 수 있으므로, 저장 후 diff로 한 번 더 확인.
3. **`filters/keyword_filter.py`** 는 한 번에 너무 많은 키워드를 넣으면 정상 기사까지 걸러질 수 있어, **예시 URL/제목**을 남기고 주석으로 의도를 적어두면 이후 담당자에게 도움이 됩니다.
4. CI에서 **필터 검증 스텝**이 있습니다. `EXCLUDE_KEYWORDS`에 특정 문자열이 반드시 있어야 한다는 검사이므로, 리스트를 대폭 정리할 때는 `.github/workflows/news-clipping.yml`의 해당 `python -c "..."` 도 함께 조정해야 할 수 있습니다.
5. **비밀번호·API 키는 절대 커밋하지 않기.** `.env`를 쓰더라도 `.gitignore`에 포함되는지 확인.

## 11. 자주 겪는 문제

| 증상 | 점검 |
|------|------|
| 메일에 내 수정이 반영 안 됨 | 로컬이면 저장·경로 확인. Actions면 **푸시 여부**와 **실행된 커밋 SHA** 확인. |
| 수집이 0건 | `last_send_at`이 최근으로 잡혀 실제로는 새 기사가 없을 수 있음. 로컬에서 `data/last_send_at.txt` 백업 후 삭제해 범위를 넓혀볼 수 있음(주의: 중복 발송 범위가 넓어짐). |
| SMTP 실패 | `SENDER_TO`, 앱 비밀번호, `SMTP_USER`/`SENDER_FROM` 일치 여부. |
| LLM 요약 실패 | `summarizer.yaml`의 provider와 환경변수 키 이름이 맞는지. |
| import 오류 | 터미널 cwd가 **프로젝트 루트**인지, `python run_batch.py`처럼 루트에서 실행하는지 확인. |
